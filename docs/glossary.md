# KinderHub Glossary

## Core Concepts

### Activity
A bookable event or class for children offered by a provider. Examples: swimming lesson, art class, coding workshop.

### Booking
A confirmed reservation for an activity, including date, time, child, and payment status.

### Provider
An individual or organization that offers activities through KinderHub.

### Parent
A registered user who discovers and books activities for their children.

### Child Profile
Information about a child (name, age, interests) used for activity recommendations and booking.

---

## Activity Types

### One-Time
Single session activities (e.g., workshop, event).

### Recurring
Regular sessions (e.g., weekly swimming class).

### Camp
Multi-day programs (e.g., summer camp).

### Drop-In
Activities that don't require advance booking.

---

## Booking States

| State | Description |
|-------|-------------|
| Pending | Booking created, awaiting payment |
| Confirmed | Payment received, booking confirmed |
| Cancelled | Booking cancelled by parent or provider |
| Completed | Activity attended |
| No-Show | Child did not attend |

---

## Technical Terms

### KMP
Kotlin Multiplatform — technology enabling shared code across Android, iOS, and Web.

### Compose
Jetpack Compose / Compose Multiplatform — declarative UI framework.

### Auth0
Third-party authentication service used for user login.

### WASM
WebAssembly — technology enabling Kotlin code to run in web browsers.

---

## Business Terms

### GMV
Gross Merchandise Value — total value of bookings processed.

### Take Rate
Percentage of GMV retained as revenue.

### MAU
Monthly Active Users.

### NPS
Net Promoter Score — measure of customer satisfaction.

---

## Roles

### Admin
KinderHub team member with full platform access.

### Provider Admin
Provider account owner with full provider access.

### Provider Staff
Provider team member with limited access.

### Parent
Standard user account for booking activities.
